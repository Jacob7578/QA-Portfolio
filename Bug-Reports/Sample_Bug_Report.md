SAMPLE BUG REPORT - created as a blueprint for writing bug reports. Created using available internet resources on how to write a professional bug report.

✔️ Bug ID

BUG-001

✔️ Title:

The login button does not respond after entering valid credentials.

✔️ Environment:

- Application: Demo Abstract Login Page
- Version: 1.0
- Operating System: Windows 11
- Browser: Google Chrome 138
- Tester: Jakub M
- Date: 27-07-2026

✔️ Severity

High

✔️ Priority

High

✔️ Preconditions

- User is on the login page
- User has a valid account
- Internet connection is stable

✔️ Steps to Reproduce

1. Open the Login page.
2. Enter a valid email address.
3. Enter the correct password.
4. Click the "Login" button.

✔️ Expected Result:

The user should be successfully logged in and redirected to the Home page.

✔️ Actual Result:

Nothing happens after clicking the "Login" button.
The user stays on the Login page without any message or loading indicator.

✔️ Reproducibility:

100% 

✔️ Attachments:

Screenshot: login_button_bug.png

✔️ Notes:

The issue occurs only in Google Chrome.
Firefox works correctly.




About the structure: Each section has its own task.

Bug ID               - number of a bug
Title                - short description of a problem
Environment          - Where was the test done? (Chrome, Opera, Android, iOS)
Severity             - How serious the bug is.
Priority             - How fast should it be fixed.
Preconditions        - What conditions must be met before the test.
Steps to Reproduce   - Precise steps leading to the bug.
Expected Result      - What should happen.
Actual Result        - What has really happened.
Reproducibility      - Is the bug occurring always?
Attachments          - Screenshots, videos, logs.
Notes                - Additional information.
